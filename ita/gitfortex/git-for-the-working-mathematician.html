Cos'è git?

Wikipedia

Git è un software di controllo versione distribuito utilizzabile da interfaccia a riga di comando, creato da Linus Torvalds nel 2005.

Si tratta di un software che gestisce automaticamente la storicizzazione ed il versionamento dei file contenuti in una cartella.

Persistent List (parte I)

List

Per imparare ad usare Git bisogna capire che struttura di dati astratta utilizza al suo interno.

Una lista è una sequenza finita, ordiata e omogenea di elementi.

Ordinata: gli elementi sono ordinati con un indice.

Omogenea: gli elementi sono dello stesso "tipo".

In una normale lista possiamo:

  • Aggiungere un elemento in coda
  • Leggere l'elemento all'indice i
  • Modificare l'elemento all'indice i
  • Eliminare l'elemento all'indice i

Persistent

Una lista persistente le operazioni possibili sono solamente:

  • Aggiungere un elemento in coda
  • Leggere l'elemento all'indice i, ottenendone una copia

Storicizzazione

Per la storicizzazione le liste persistenti sono ottime, le passate versioni non devono mai essere modificate.

L'elemento in coda rappresenta l'elemento attivo.

Se si vuole ritornare ad una versione precedente d'indice i basta leggere l'elemento e aggiungerlo alla coda della lista.

Puramente funzionale

Come vedremo in seguito è importante capire che queste liste sono puramente funzionali.

In una lista classica le operazioni modificano la lista stessa: se ho una lista \(l\) di \(n\) elementi, se ne elimino uno \(l\) sarà una lista con \(n-1\) elementi.

In una lista puramente funzionale le operazioni sono funzioni che creano una nuova lista: se ho una lista \(l_1\) di \(n\) elementi, se invoco la funzione per eliminarne uno otterrò una nuova lista \(l_2\) con \(n-1\) elementi, mentre la prima lista \(l_1\) resterà invariata.

Storicizzare i documenti

La branch master

Git lavora con le cartelle, data una cartella potrà gestirà il versionamento dei file presenti in una cartella (ed in tutto l'albero delle sotto cartele).

Una volta installato git l'operazione per fargli gestire una cartella è git init, la cartella ed i suoi file rimarranno invariati e funzioneranno come prima, l'unica differenza sarà la creazione di un file .git

La creazione, la modifica o il salvataggio di un file non implicano automaticamente un versionalento di git. Questo deve essere domandato esplicitamente.

Se si crea un file che si vuol far gestire da git bisogna utilizzare il comando git add

Se si modifica o elimina un file gestito da git e si vuole storicizzare tale cambiamento bisogna utilizzare il comando git commit, aggiungendo un messaggio per indicare la motivazione di tale cambiamento.

Queste operazioni corrispono ad aggiungere un elemento ad una persistent list che prende il nome di master branch

Se si vuole ottenere una copia di un precedente elemento (commit) bisogna utilizzare il comando git revert, un successivo git commit lo renderà la versione corrente.

Persistent List (parte II)

Una lista è un ramo di un albero

Come già detto precedentemente una lista persistente è una struttura di dati puramente funzionale.

Sia \(l\) una lista, se aggiungiamo un elemento a questa lista otterremo una nuova lista \(l_1\) lascianod \(l\) invariata.

Se aggiungiamo un altro elemento a \(l\) una nuova lista \(l_2\).

Le liste \(l_1\), \(l_2\) e tutte coloro che creeremo aggiungendo nuovi elementi sono tutte diverse ma condividono il segmento \(l\), sono i rami di un albero.

L'albero delle versioni

Questa struttura ci è utile per descrivere le diverse versioni di uno stesso documento che possono essere modificate in parallelo.
